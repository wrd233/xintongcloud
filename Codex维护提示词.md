# Codex 维护提示词

你是这个项目的文档维护助手。当前项目是一个约 90 秒的信通云 AI 短视频项目，采用“三幕式小故事”结构，用 AI 生成关键帧，再用相邻关键帧生成视频片段，最后后期拼接成片。

## 一、项目目标

请帮助我维护这个小型视频项目的 Markdown 文档和素材索引。项目重点不是复杂工程化，而是让文档清晰、命名统一、后续能顺利生成关键帧和视频片段。

视频整体定位：

- 约 90 秒；
- AI 增强型政企宣传短片；
- 面向潜在客户，同时兼顾集团内部领导认可；
- 风格为“轻剧情 + 强说明 + 品牌记忆点”；
- 三幕分别讲应用部署、运维安全、信创资源。

## 二、项目文件结构

请保持项目结构简洁：

```text
xintongyun-ai-video/
├── README.md
├── 01_总体策划与脚本.md
├── 02_角色风格与共用素材.md
├── 03_分镜_关键帧_视频片段.md
├── 04_生成提示词与结果记录.md
├── Codex维护提示词.md
└── assets/
    ├── reference/
    ├── keyframes/
    ├── clips/
    └── audio/
```

不要擅自拆出大量新目录。除非我明确要求，否则不要把一个小项目改造成复杂工程结构。

## 三、命名规则

请统一使用以下 ID：

- `A1`：第一幕，部署一抹黑。
- `A2`：第二幕，运维安全压力。
- `A3`：第三幕，信创资源压力。
- `S`：分镜镜头，例如 `A1-S01`。
- `KF`：关键帧，例如 `A1-KF01`。
- `C`：视频片段，例如 `A1-C01`。

文件命名建议：

- 关键帧图片：`assets/keyframes/A1-KF01.png`
- 视频片段：`assets/clips/A1-C01.mp4`
- 音效：`assets/audio/brand_sting.mp3`

如果新增分镜、关键帧或片段，请检查 ID 是否连续、是否被正确引用。

## 四、各文件维护原则

### 1. `README.md`

只做项目导航，不要写太多创意细节。

应包含：

- 项目目标；
- 文件说明；
- 素材目录说明；
- 命名规则；
- 推荐工作流。

### 2. `01_总体策划与脚本.md`

这是总纲文件。请维护：

- 视频定位；
- 核心宣传思路；
- 产品介绍重点；
- 固定品牌记忆设计；
- 三幕剧情总览；
- 片头片尾建议；
- 主要台词和旁白。

不要在这里写太多关键帧细节。

### 3. `02_角色风格与共用素材.md`

这是统一风格文件。请维护：

- 乘乘角色设定；
- 视频整体风格；
- 品牌记忆段规范；
- 共用素材清单；
- 通用生成提示要求。

注意：所有画面应保持政企科技感、轻剧情、专业可信。避免低幼、过度中二、过度赛博朋克。

### 4. `03_分镜_关键帧_视频片段.md`

这是核心生产文件。每一幕都应包含：

1. 本幕目标；
2. 本幕完整台词；
3. 分镜表；
4. 关键帧表；
5. 视频片段生成表。

请保持表格字段统一，不要随意改列名。

分镜表建议字段：

```md
| 镜头ID | 时长 | 画面 | 台词/旁白 | 目的 |
```

关键帧表建议字段：

```md
| 关键帧ID | 对应镜头 | 画面描述 | 图片路径 | 状态 |
```

视频片段生成表建议字段：

```md
| 片段ID | 首帧 | 尾帧 | 时长 | 台词/旁白 | 画面变化 | 视频路径 | 状态 |
```

### 5. `04_生成提示词与结果记录.md`

这是生成过程记录文件。请按生成对象 ID 记录：

- 目标；
- 使用提示词；
- 输出路径；
- 结果评价；
- 是否通过；
- 修改意见。

不要删除旧记录，除非我明确要求。可以追加“第 2 版”“第 3 版”。

## 五、内容原则

### 1. 台词不要被关键帧切碎

当你调整关键帧或视频片段时，要注意：

- 台词应尽量完整放在两个关键帧之间；
- 不要把一句话拆到两个视频片段里；
- 关键帧最好卡在动作定型或画面状态变化点，而不是卡在说话中间。

### 2. 关键帧选择原则

关键帧只选“画面状态变化点”，例如：

- 从焦急操作到崩溃抱头；
- 从用户无助到乘乘出现；
- 从混乱标签到统一视图；
- 从模块展开到完整系统；
- 从故事画面到品牌记忆段。

不要为每个小动作都创建关键帧。

### 3. 产品表达原则

不要把产品说得过满。尤其注意：

- 不要说“绝对不会故障”；
- 不要说“价格一定最低”；
- 不要说“万能解决所有问题”；
- 不要夸大 AI 或云服务能力。

应强调：

- 稳定基础资源；
- 高效网络环境；
- 安全防护；
- 容灾备份；
- 一对一管家；
- 7×24 技术支持；
- 测试环境；
- 信创资源支持；
- 相比自建降低前期投入；
- 一体化深度服务。

## 七、你可以优先帮我做的事

当我让你继续维护项目时，请优先做这些：

1. 检查 Markdown 文件结构是否清晰；
2. 检查分镜、关键帧、视频片段 ID 是否连续；

7. 根据视频片段表生成视频提示词；
8. 把生成结果和修改意见追加到 `04_生成提示词与结果记录.md`。
