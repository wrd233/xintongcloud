# 信通云 AI短视频项目

本项目用于制作一支约 90 秒的信通云 AI 宣传短片。视频采用“三幕式小故事”结构，围绕基层单位在应用部署、运维安全、信创资源方面的典型痛点展开，通过“乘乘”这一 IP 角色完成问题引入、能力介入和品牌记忆的表达。

## 项目目标

制作一支适合潜在客户观看、同时能够获得集团内部领导认可的 AI 增强型政企宣传短片。

整体风格应做到：

- 有故事感，但不过度搞笑；
- 有 AI 生成特色，但不炫技；
- 有产品卖点，但不是硬念参数；
- 有传播记忆点，同时保持稳重可信。

## 文件说明

- `01_总体策划与脚本.md`：记录视频整体创意、三幕结构、完整台词和旁白。
- `02_角色风格与共用素材.md`：记录乘乘角色设定、信通云 logo、品牌记忆段、视觉风格和素材清单。
- `03_分镜_关键帧_视频片段.md`：记录每一幕的分镜、关键帧、关键帧之间的视频生成计划。
- `04_生成提示词与结果记录.md`：记录实际使用的图片/视频生成提示词、生成结果、修改意见。
- `Codex维护提示词.md`：给 Codex 使用的维护提示词，用于后续持续整理和更新本项目文档。

## 素材目录

- `assets/reference/`：参考图，例如乘乘原始形象、信通云 logo、风格参考图。
- `assets/keyframes/`：生成出的关键帧图片，建议用 `A1-KF01.png` 这类命名。
- `assets/clips/`：生成出的视频片段，建议用 `A1-C01.mp4` 这类命名。
- `assets/audio/`：品牌音效、配音、背景音乐等音频素材。

## 命名规则

三幕编号：

- `A1`：第一幕，部署一抹黑。
- `A2`：第二幕，运维安全压力。
- `A3`：第三幕，信创资源压力。

对象编号：

- `S`：分镜镜头，例如 `A1-S01`。
- `KF`：关键帧，例如 `A1-KF01`。
- `C`：视频片段，例如 `A1-C01`。

## 推荐工作流

1. 先确认 `01_总体策划与脚本.md` 的三幕创意和台词。
2. 再确认 `02_角色风格与共用素材.md` 的统一角色、风格和品牌记忆段。
3. 在 `03_分镜_关键帧_视频片段.md` 中逐幕完善分镜、关键帧和片段生成计划。
4. 先跑通第一幕，从关键帧图片生成到视频片段生成。
5. 生成过程中的提示词、结果和修改意见记录到 `04_生成提示词与结果记录.md`。
6. 第一幕稳定后，再复制方法推进第二幕、第三幕。
